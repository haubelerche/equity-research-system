## [Unreleased]

### Added
- **T10** `backend/app/dataops/quality_checks.py`: Pure rule-based data quality checks with no LLM or DB calls. Implements `QualityCheckResult` dataclass and three check functions: `check_completeness` (verifies 8 default required `line_item_code`s are present; returns one failed result per missing code), `check_value_ranges` (detects negative revenue as error, implausible net income outside `[-revenue, revenue]` as warning, negative implied equity as error), `check_balance_sheet_equation` (verifies `assets ≈ liabilities + equity` within 1% tolerance; returns info/skipped when any operand is missing). `run_all_checks` combines all three. `fact_ids` in each result collects affected row UUIDs.
- **T10** `backend/app/repositories/quality_repository.py`: Supabase write layer for `governance.data_quality_checks`. `save_quality_checks` maps `QualityCheckResult` fields to DB columns (`message`, `affected_record_ids` as string list, `check_type`, `severity`, `status`). Maps `"warning"` status to `"failed"` to satisfy DB constraint (`passed|failed|skipped`). Returns count inserted; returns 0 on empty input.
- **T10** `backend/app/dataops/job_runner.py`: Extended `_store_financial` to call `run_all_checks` and `save_quality_checks` after `insert_fact_lineage`. Added imports for both new modules.
- **T10** `backend/tests/unit/test_quality_checks.py`: 22 pure unit tests covering all three check functions and `run_all_checks`. Tests use realistic DHG-like VND values. Covers: completeness all-present/missing/custom codes, period label formatting (annual + quarterly), range negative revenue/normal data/negative equity/implausible net income/fact_id collection, balance equation holds/within-tolerance/10%-off-fails/missing-value-skipped/fact_ids-on-failure, and combined run_all_checks count + quarterly label propagation.
- **T10** `backend/tests/unit/test_quality_repository.py`: 7 unit tests using MagicMock for the Supabase client. Covers: insert called on correct table, empty input fast-return, fact_ids serialized to strings, column mapping correctness, multiple results produce unique check_ids, warning status mapped to failed, return count equals input length. Total unit test count: 180 (was 151).
- **T09** `backend/app/repositories/canonical_repository.py`: Supabase write layer for canonical tables. Implements `upsert_financial_facts` (ON CONFLICT on 7-column composite key), `upsert_prices` (ON CONFLICT on ticker/trade_date/source_id), `upsert_snapshot`, and `upsert_profile`. All functions serialize UUID fields to `str` and `date` fields to ISO strings via a shared `_serialize_row` helper before calling supabase-py `.upsert()`. Returns row count; returns 0 immediately on empty input.
- **T09** `backend/app/repositories/governance_repository.py`: Supabase write layer for `governance.fact_lineage`. Implements `insert_fact_lineage` with ON CONFLICT DO NOTHING on (fact_id, source_id, ingestion_run_id). Same UUID/date serialization pattern as canonical_repository.
- **T09** `backend/app/dataops/lineage.py`: Pure function `build_lineage_rows` that maps a list of `CanonicalFactRow` objects + `raw_payload_id` to `FactLineageRow` objects. Generates `uuid4()` per row as `fact_id` (no `fact_id` field on `CanonicalFactRow`). Returns empty list on empty input; supports configurable `transform_version`.
- **T09** `backend/app/dataops/job_runner.py`: Extended `process_job` to capture `source_id` from `register_source` return value and call `_normalize_and_store` (new private function) after payload storage. `_normalize_and_store` routes by `job_type` to the appropriate normalizer, then calls the matching canonical/governance repository write functions. Handles all 5 job types: `financial_statement_refresh` (normalize + upsert facts + insert lineage), `daily_price_refresh` (normalize + upsert prices), `market_snapshot_refresh` (normalize + upsert snapshot if not None), `company_profile_refresh` (normalize + upsert profile if not None), `news_refresh` (DEBUG log, skipped).
- **T09** `backend/tests/unit/test_canonical_repository.py`: 11 unit tests using MagicMock for the Supabase client. Covers: upsert called with correct table names, UUID/date fields serialized to strings, empty-list fast-return (no DB call), correct `on_conflict` column sets for all four write functions, and multi-row count return.
- **T09** `backend/tests/unit/test_lineage.py`: 9 unit tests (pure, no mocking). Covers: N rows → N lineage rows, correct `raw_payload_id` propagation, empty input, default and custom `transform_version`, source_id and ingestion_run_id passthrough, UUID type check on fact_id, and FactLineageRow instance check. Total unit test count: 151 (was 131).
- **T08** `backend/app/dataops/normalization.py`: Pure transformation module (no LLM, no DB calls) that maps raw Vnstock payloads to typed Pydantic rows. Implements `load_taxonomy` (builds a `(statement_type, vi_name_lower)` → `line_item_code` lookup from the taxonomy YAML with module-level cache), `normalize_financial_statement` (flat-dict iteration, float coercion, skips unknown keys and non-numeric values), `normalize_price_data` (OHLCV with ISO date parsing and adjusted-close fallback chain), `normalize_snapshot` (multi-key market_cap extraction; returns `None` when missing), and `normalize_profile` (multi-key company_name; defaults for exchange/industry/subsector; returns `None` when name absent).
- **T08** `backend/tests/unit/test_normalization.py`: 16 unit tests covering: taxonomy primary-vi and alias lookups, financial statement happy path / unknown key / non-numeric value / empty input, price data valid OHLCV / missing date / `close_adj` preference, snapshot valid / missing market_cap / `marketCap` camelCase key, profile valid / missing name / `shortName` fallback, and quarterly period preservation. All 16 tests pass (0.17s, no database or network calls).
- **T07** `data/taxonomy/financial_line_items.yaml`: Taxonomy YAML (version v1) mapping Vnstock raw Vietnamese column names to internal `line_item_code` for all three statement types: `income_statement` (11 items), `balance_sheet` (15 items), `cash_flow` (6 items). Each entry has a primary `vi` name and `aliases` list covering alternate spellings observed in Vnstock output. No Supabase, Vnstock, or LLM imports.
- **T07** `backend/app/schemas/__init__.py`: Empty package init for the new schemas module.
- **T07** `backend/app/schemas/financials.py`: Pydantic v2 models for all canonical row types used by normalization and repository layers: `CanonicalFactRow`, `PriceRow`, `SnapshotRow`, `ProfileRow`, `FactLineageRow`, `FinancialRatioRow`, `PeerMetricRow`. `FinancialRatioRow.value` is `float | None` to support zero-denominator cases. All Literal fields use v2 syntax; no `class Config` blocks.
- **T07** `backend/tests/unit/test_schemas_financials.py`: 7 unit tests covering valid construction, Literal-field rejection for invalid `statement_type` and `quality_status`, `None` value on `FinancialRatioRow`, empty `input_fact_ids` default, nullable OHLCV fields, and `PeerMetricRow` defaults. Total unit test count: 108 (was 101).
- **T06** `backend/app/dataops/job_runner.py`: Job runner that polls queued ingestion jobs, acquires optimistic locks, creates ingestion runs, dispatches to the correct `vnstock_client` fetch function via `JOB_DISPATCH_MAP`, saves raw payloads, registers source manifest entries, and updates job/run status on success or failure. No normalization (Phase 2). Implements `process_job`, `run_worker_loop`, and `_build_fetch_kwargs`.
- **T06** `backend/app/dataops/__init__.py`: Empty package init.
- **T06** `backend/tests/unit/test_job_runner.py`: 24 unit tests covering lock-fail fast-return, happy-path steps (create run, correct fetch dispatch, save payload, register source, success status updates), error-path (failed status, re-raise), unknown job type, record count for list vs dict data, kwargs construction for all job types, and worker loop iteration/error-tolerance.
- **T06** `backend/tests/conftest.py`: Suite-wide `sys.modules` stub for `vnstock` package (not installed in test env); unblocks `vnstock_client.py` imports during unit testing and fixes pre-existing collection failures in `test_vnstock_client.py`.
- **T05** `backend/app/connectors/vnstock_client.py`: Single adapter that is the ONLY permitted import site for `vnstock`. Implements `fetch_company_profile`, `fetch_price_history`, `fetch_market_snapshot`, `fetch_financial_statements`, and `fetch_news_events`. All methods return a typed envelope with `ticker`, `source_endpoint`, `request_params`, `retrieved_at`, and `data`. `fetch_news_events` never raises. `CONNECTOR_VERSION = "vnstock-0.3"`, `PARSER_VERSION = "v1"`.
- **T05** `backend/tests/unit/test_vnstock_client.py`: 27 unit tests covering all five connector methods plus AST-based guard tests confirming `raw_repository.py`, `ops_repository.py`, and `source_registry.py` contain no direct `vnstock` imports. All tests pass (mocked, no real API calls).
